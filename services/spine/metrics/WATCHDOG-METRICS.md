## The promotion rule

The forest serves only while it beats the fixed threshold on F1 over held out
labelled days. The rule lives in `src/watchdog/eval.ts` and serving code
reads only the verdict in `watchdog-metrics.json`; nothing else decides.

## Verdict

The forest beat the fixed threshold on F1 over 22 labelled leak days, so forest:v1 serves. The threshold never fired because the injected leakage hides inside one kombi's takings: a heavy skim on one of four vehicles moves the route total by well under the 40 percent drop the threshold needs. That dilution is the real world case, and it is exactly the argument for a model over a rule here.

Flags always describe patterns (a day, a route, an unnamed vehicle), never a
person. That rule is enforced by unit test on the explanation templates.
